# Optivoy Optimizer 部署说明

最后更新：2026-03-30

## 一、先检查本地是否可访问 PyPI

先执行（排除本机代理影响）：

```bash
env -u http_proxy -u https_proxy -u all_proxy -u HTTP_PROXY -u HTTPS_PROXY -u ALL_PROXY \
curl -I --max-time 8 https://pypi.org/simple/ortools/
```

如果失败，可在构建时指定镜像源（例如阿里云）：

```bash
--build-arg PIP_INDEX_URL=https://mirrors.aliyun.com/pypi/simple/
```

## 二、镜像构建与打包（本地）

```bash
cd /Users/code-2026/optivoy.top/optivoy-optimizer
docker buildx build --platform linux/amd64 -t optivoy-optimizer:prod --load .
docker save optivoy-optimizer:prod | gzip > optivoy-optimizer-prod.tar.gz
```

如果直连 PyPI 不通，改用：

```bash
cd /Users/code-2026/optivoy.top/optivoy-optimizer
docker buildx build \
  --platform linux/amd64 \
  -t optivoy-optimizer:prod \
  --build-arg PIP_INDEX_URL=https://mirrors.aliyun.com/pypi/simple/ \
  --load .
docker save optivoy-optimizer:prod | gzip > optivoy-optimizer-prod.tar.gz
```

## 三、上传到 OSS

```bash
ossutil cp optivoy-optimizer-prod.tar.gz oss://optivoy/docker-images/optivoy-optimizer-prod.tar.gz
```

## 四、服务器下载与加载

```bash
ossutil cp oss://optivoy/docker-images/optivoy-optimizer-prod.tar.gz /opt/optivoy/releases/
sudo docker load < /opt/optivoy/releases/optivoy-optimizer-prod.tar.gz
```

## 五、服务器启动（独立 compose）

先创建共享内网：

```bash
sudo docker network create optivoy-internal || true
```

把当前目录的 `docker-compose.server.yml` 上传到服务器：

```bash
scp -i ~/.ssh/admin.pem -P 22 /Users/code-2026/optivoy.top/optivoy-optimizer/docker-compose.server.yml root@60.205.92.35:/opt/optivoy/app/docker-compose.optimizer.yml
```

启动：

```bash
cd /opt/optivoy/app
sudo docker compose -f docker-compose.optimizer.yml up -d
sudo docker compose -f docker-compose.optimizer.yml ps
sudo docker logs --tail 100 optivoy-optimizer
```

## 七、与 backend 分开 compose 的内网调用

backend 容器只要也加入 `optivoy-internal` 网络，就可以固定访问：

```text
http://optivoy-optimizer:8088/solve
```

建议在 backend compose 中配置：

```env
OPTIMIZER_BASE_URL=http://optivoy-optimizer:8088
```

## 八、关于“是否需要额外配置才能运行 CP-SAT”

需要。镜像只是运行环境，核心在于业务建模输入：

- 点位停留时长
- 每日可游玩时间窗（强度）
- 酒店策略
- 优化目标
- 距离/通勤矩阵

这些必须在调用 `/solve` 时传入，或者由 backend 在调用前计算好并传入。



本地构建运行方案
docker build -t optivoy-optimizer:local .
docker run -d --name optivoy-optimizer-local -p 8088:8088 --restart unless-stopped optivoy-optimizer:local
curl http://127.0.0.1:8088/health

## 1. 重构背景与目标

### 核心问题

当前 optimizer 存在以下根本性缺陷（详见《优化器评估与问题分析报告.md》）：

1. **CP-SAT 把分天与配酒店耦合求解**，产生 O(n×d×k) 三维变量，2.5 秒内难以求得真正最优解
2. **`point_day_load` 未扣除交通时间和餐饮 overhead**，导致 CP-SAT 系统性高估每天容量
3. **餐饮约束是硬异常**，全排列失败后直接 HTTP 400，无任何恢复机制
4. **购物中心、主题乐园等自带餐饮的点位**被视为无餐饮，强制插入虚拟餐却因时间不足插不进去

### 重构目标

- 用**三阶段简单算法**替换 CP-SAT 联合求解，代码量从 1593 行压缩到 ~850 行
- 分天结果不再被酒店位置"污染"，更符合地理直觉
- 酒店分配独立成一步，逻辑清晰可测试
- 修复所有 P0/P1 餐饮约束 bug
- 支持"自带餐饮"场景（购物中心、主题乐园等）

## 3. 新增字段：自带餐饮标记

### 背景

北京SKP、国贸商城、环球影城、迪士尼等点位均有大型美食广场或内部餐厅。
游客在这类点位游玩时会在内部用餐，无需单独安排餐厅，也无需插入"虚拟餐"。
当前系统不感知这一事实，强制插入 45 分钟虚拟餐时间，导致时间预算超标。

### 新增字段

```sql
-- shopping_places：大型购物中心、商业综合体
ALTER TABLE shopping_places ADD COLUMN "hasFoodCourt" boolean NOT NULL DEFAULT false;

-- spots：主题乐园、大型景区（如故宫内有餐厅、环球影城等）
ALTER TABLE spots ADD COLUMN "hasFoodCourt" boolean NOT NULL DEFAULT false;
```

### 语义

| `hasFoodCourt` | 含义 | Optimizer 行为 |
|----------------|------|---------------|
| `false`（默认）| 普通景点/商城，无餐饮 | 按原逻辑，需插虚拟餐或安排专门餐厅 |
| `true` | 内部有餐饮，游客会在此用餐 | 视为可满足 `lunch` 和 `dinner`，等同于餐厅功能 |

### Phase 1：地理聚类分天

#### 有效日容量计算

```python
TRAVEL_OVERHEAD_PER_POINT = 30   # 每个点平均增加 30 分钟交通开销
MEAL_OVERHEAD_MINUTES     = 45   # 每顿虚拟餐 45 分钟

def effective_day_capacity(req: SolveRequest, n_points: int, has_food: bool) -> int:
    """
    从 pace 总分钟数中扣除：
      1. 每顿所需虚拟餐时间（仅当当天无自带餐饮点位时）
      2. 点位间交通开销估算（含返回酒店）
    """
    base  = PACE_MODE_WINDOWS[req.paceMode][2]          # 660 / 780 / 480

    # 扣餐饮 overhead
    if req.mealPolicy == "auto" and not has_food:
        meals = len(required_meal_slots(req))            # light=1, standard/compact=2
        base -= meals * MEAL_OVERHEAD_MINUTES

    # 扣交通 overhead：n_points 段点间 + 1 段返回酒店
    base -= (n_points + 1) * TRAVEL_OVERHEAD_PER_POINT

    return max(120, base)
```

**修正效果对比（standard，3 个非餐厅点）**：

| 项目 | 旧公式 | 新公式 |
|------|--------|--------|
| 基础预算 | 660 min | 660 min |
| 扣餐饮（2×45）| ✗ 忽略 | −90 min |
| 扣交通（4段×30）| ✗ 忽略 | −120 min |
| **有效容量** | **660 min** | **450 min** |
| 3 点实际负荷（3×192）| 576 ≤ 660 ✓ 错误通过 | 576 > 450 ✗ 正确拒绝 |

#### 贪心聚类算法

```python
def cluster_points_to_days(
    req: SolveRequest,
    points: list[PointPrepared],
    lookups: LookupMaps,
    windows: list[DayWindow],
) -> list[list[int]]:
    """
    贪心地理聚类：
      1. 从未分配点中选一个"种子点"（离当前未分配集合重心最近）
      2. 不断向当天加入距当前聚类重心最近的点，直到容量不足
      3. 跳过当天不可用的点（营业时间限制）
      4. 换到下一天，重复

    时间复杂度：O(n² × d)，对 n≤50 完全够用。
    """
    unassigned = list(range(len(points)))
    day_clusters: list[list[int]] = []

    for window in windows:
        if not unassigned:
            break

        cluster: list[int] = []
        cluster_load = 0
        # 过滤出今天不可用的点，推迟到后续天
        available_today = [
            i for i in unassigned
            if point_can_be_on_day(points[i].point, window)
        ]
        if not available_today:
            continue

        # 选种子：离所有未分配点平均重心最近的点
        seed = pick_seed(available_today, points, lookups)
        cluster.append(seed)
        unassigned.remove(seed)
        cluster_load += point_day_load(points[seed].point, window.date_value)

        # 贪心扩展
        while True:
            candidates = [
                i for i in unassigned
                if point_can_be_on_day(points[i].point, window)
            ]
            if not candidates:
                break

            has_food = any(points[i].point.hasFoodCourt for i in cluster)
            capacity = effective_day_capacity(req, len(cluster) + 1, has_food)
            next_i = pick_nearest_to_cluster(candidates, cluster, points, lookups)
            next_load = point_day_load(points[next_i].point, window.date_value)

            if cluster_load + next_load > capacity:
                break  # 今天装不下了，明天再说

            cluster.append(next_i)
            unassigned.remove(next_i)
            cluster_load += next_load

        if cluster:
            day_clusters.append(cluster)

    if unassigned:
        # 剩余点无法在 maxDays 内安排
        raise HTTPException(
            status_code=400,
            detail={
                "code": "POINTS_EXCEED_MAX_DAYS",
                "message": f"{len(unassigned)} point(s) cannot fit within {req.maxDays} days",
                "remainingPointIds": [points[i].point.id for i in unassigned],
            },
        )

    return day_clusters


def pick_seed(candidates: list[int], points: list[PointPrepared], lookups: LookupMaps) -> int:
    """选距所有候选点驾车时间总和最小的点作为种子"""
    best_i, best_cost = candidates[0], float("inf")
    for i in candidates:
        total = sum(
            directed_driving_minutes(
                points[i].point.id, points[i].departure_coord,
                points[j].point.id, points[j].arrival_coord,
                lookups,
            )
            for j in candidates if j != i
        )
        if total < best_cost:
            best_cost, best_i = total, i
    return best_i


def pick_nearest_to_cluster(
    candidates: list[int],
    cluster: list[int],
    points: list[PointPrepared],
    lookups: LookupMaps,
) -> int:
    """选与当前聚类中所有点平均驾车时间最短的候选点"""
    best_i, best_cost = candidates[0], float("inf")
    for i in candidates:
        avg = sum(
            directed_driving_minutes(
                points[c].point.id, points[c].departure_coord,
                points[i].point.id, points[i].arrival_coord,
                lookups,
            )
            for c in cluster
        ) / len(cluster)
        if avg < best_cost:
            best_cost, best_i = avg, i
    return best_i
```

### Phase 2：质心配酒店

```python
def assign_hotel_per_day(
    day_clusters: list[list[int]],
    points: list[PointPrepared],
    hotels: list[HotelPrepared],
    lookups: LookupMaps,
    req: SolveRequest,
) -> list[int]:
    """
    为每天的点集分配最优酒店。
    评分标准：该酒店到当天所有点的平均往返驾车时间（越小越好）。

    hotelMode = "single"：所有天用同一家酒店（选全程最优）。
    hotelMode = "multi"：每天独立选最近酒店（相邻两天距离近时自动复用同一家）。
    """
    if req.hotelMode == "single":
        all_point_indexes = [i for cluster in day_clusters for i in cluster]
        best = find_nearest_hotel(all_point_indexes, points, hotels, lookups)
        return [best] * len(day_clusters)

    hotel_per_day: list[int] = []
    for cluster in day_clusters:
        best = find_nearest_hotel(cluster, points, hotels, lookups)
        hotel_per_day.append(best)
    return hotel_per_day


def find_nearest_hotel(
    point_indexes: list[int],
    points: list[PointPrepared],
    hotels: list[HotelPrepared],
    lookups: LookupMaps,
) -> int:
    """选平均往返驾车时间最小的酒店"""
    best_h, best_cost = 0, float("inf")
    for h, hotel in enumerate(hotels):
        total = 0.0
        for i in point_indexes:
            to_p = directed_driving_minutes(
                hotel.hotel.id, hotel.departure_coord,
                points[i].point.id, points[i].arrival_coord,
                lookups,
            )
            from_p = directed_driving_minutes(
                points[i].point.id, points[i].departure_coord,
                hotel.hotel.id, hotel.arrival_coord,
                lookups,
            )
            total += to_p + from_p
        avg = total / len(point_indexes)
        if avg < best_cost:
            best_cost, best_h = avg, h
    return best_h
```

#### 修复 3：虚拟餐软约束（不再硬抛异常）

```python
MEAL_MISS_PENALTY_MINUTES = 120  # 错过餐窗的惩罚分钟当量（加入 effective_cost）

def maybe_insert_virtual_meal_soft(
    current_minutes: int,
    slot: str,
    satisfied: set[str],
    day_end_minutes: int,
) -> tuple[int, bool, float]:
    """
    尝试插入虚拟餐。
    返回：(new_current_time, inserted, penalty)
    - 可插入：返回餐后时间，penalty=0
    - 不可插入：返回原时间，penalty=MEAL_MISS_PENALTY_MINUTES（不再 raise）
    """
    if slot in satisfied:
        return current_minutes, False, 0.0
    slot_start, slot_end = MEAL_SLOT_WINDOWS[slot]
    meal_start = max(current_minutes, slot_start)
    meal_end   = meal_start + VIRTUAL_MEAL_MINUTES
    if meal_end > min(slot_end, day_end_minutes):
        # 时间窗口不够 → 软惩罚，不抛异常
        return current_minutes, False, float(MEAL_MISS_PENALTY_MINUTES)
    satisfied.add(slot)
    return meal_end, True, 0.0
```

```python
    + float(wait_total) * 0.2
)
```

#### 修复 4：Phase 3 失败时的兜底恢复

```python
# solve_itinerary() 中调用 solve_day_route 处

try:
    route = solve_day_route(req, points, hotels, window, hotel_index,
                            cluster, start, prev_hotel, is_last, lookups)
except HTTPException as exc:
    if exc.status_code != 400:
        raise

    # 兜底 1：关闭餐饮约束重试
    req_no_meal = req.model_copy(update={"mealPolicy": "off"})
    try:
        route = solve_day_route(req_no_meal, points, hotels, window, hotel_index,
                                cluster, start, prev_hotel, is_last, lookups)
    except HTTPException:
        # 兜底 2：移走当天负荷最重的点，放到下一天
        # （将该点插回 day_clusters[day_pos+1] 开头，重新排序）
        raise HTTPException(
            status_code=400,
            detail={
                "code": "NO_FEASIBLE_DAY_ROUTE",
                "message": "no feasible itinerary for scheduled day after fallback",
                "dayNumber": day_pos + 1,
                "pointIds": [points[i].point.id for i in cluster],
                "optimizerDetail": exc.detail,
            },
        ) from exc
```
